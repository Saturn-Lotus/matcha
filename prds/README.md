## Conventions
- Each PRD has: Context, Scope, Out of scope, Data model, API surface, UI, Acceptance criteria, Security, Open questions.
- When code is built or modified against a PRD, **the PRD must be updated in the same change** (see `CLAUDE.md` → *PRD maintenance*).
- PRDs are the canonical product spec; `CLAUDE.md` is the canonical engineering-rule spec.
